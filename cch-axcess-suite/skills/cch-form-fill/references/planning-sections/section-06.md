# Section 06 — Concluding Procedures

**Forms:** KBA-105 (Sig. Estimates), KBA-102 (Engagement Completion), KBA-103 (Control Deficiencies), AUD-909 (FS Review), AUD-901 (Subsequent Events), AUD-813 (Journal Entries), AUD-814 (Related Party), KBA-902 (Review & Approval), KBA-903 (Documentation Checklist). **All are "at the conclusion" question/checklist forms — none is testwork.** The actual JE / related-party / subsequent-events testing lives in its own workpaper; these forms reference it. _(Form IDs in this file are governmental; the same numbers map to different programs on other entity titles — see `form-content-reference.md` §2b.)_

These forms get filled near the *start* (mostly correct) and revisited at the end to flip the few that changed.

---

## What you need to know to answer Section 06

### A. Aggregator / living docs — KBA-102, KBA-103, KBA-105
Near-empty on their own; **auto-populate (linked) from other forms** — significant matters, control deficiencies, and significant estimates flow *in*. Confirm defaults:
- **KBA-105:** possible management bias in estimates? → default **No.**
- **KBA-103:** control deficiencies → default **none** (else accumulate findings).
- **KBA-102:** engagement completion — significant matters, KAM (No); mostly linked; finalized last.

Knowledge: any real findings/deficiencies/significant matters (default none); otherwise nothing to gather.

### B. Reviewer checklists — KBA-902 (Review/Approval), KBA-903 (Documentation)
Large (297 / 240 boxes) but **default all "Yes / satisfied / documented."** Profile toggles up top (constants) + reviewer attestations (risk assessment performed, RMM linked to procedures, no unresolved disagreements, documentation complete & cross-referenced, rep letters & legal confirmations obtained, engagement letter, independence/ethics/acceptance documented, strategy/plan/materiality documented…).

Knowledge: **reviewer (partner/manager) names** (constant); confirm review & file complete. The few that legitimately go No/N/A: legal-counsel confirmations (no litigation → N/A), component auditor (No), KAM (No).

### C. Concluding pseudo-programs — AUD-909, AUD-901, AUD-813, AUD-814
Procedure forms, not testwork. Heavy linked auto-fill (**AUD-814 = 1,389 linked fields**). Each = entity-fact toggles + default-No findings + standard procedures signed off + **WP reference to the real testing workpaper**.
- **AUD-901 Subsequent Events:** root *"any subsequent events?"* (default No) + procedures (read interims, scan GL/journals, inquire of named management, review governing-body minutes).
- **AUD-909 FS Review:** entity-fact toggles (component units, fiduciary/proprietary funds, SI/RSI presented — from the FS) + default-No exceptions (consent request, prior-FS info, ICFR report) + analytics **Yes** + standard procedures (presentation basis, major-fund determination, fund→government-wide reconciliation). **Change in accounting principle is ASK/confirm, not a silent default-No** (a genuine material change is disclosed prominently with a prior-period restatement, but may be assessed before the FS are final — see `section-library.md` FS Review). WP ref → FS-review/analytics WP.
- **AUD-813 Journal Entries & FS Review:** controls steps **N/A** (substantive); analytics; disclosure review. WP ref → the **JE testing workpaper** (every time).
- **AUD-814 Related Party:** root *"any related-party transactions?"* (default No / from KBA-302 listing) + *"significant unusual transactions?"* (No) + disclosure. WP ref → related-party testing WP.

Knowledge: the **root findings** (default No), control approach (N/A controls), and **which testing WP index** to reference.

### D. Findings (when they exist)
Default **none**. If findings exist they are documented in KBA-103 and flow to the report. Flag findings from the findings folder / known issues; do **not** invent findings.

## Constants surfaced here (→ master list)
Reviewer (partner/manager) names; report date; **testing-WP indexes** (JE, related party, subsequent events, FS review).

## Big picture
Hundreds of boxes, almost entirely: default-Yes checklists + default-No findings + auto-linked aggregators + procedures referencing the real testing WPs. The genuine inputs reduce to: **any actual findings/deficiencies** (default none), **reviewer names**, and the **testing-WP indexes**.
