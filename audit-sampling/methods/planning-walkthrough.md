# Planning / Walkthrough

## Concept
Not a sampling method in the statistical sense. Used for planning and walkthrough procedures where the auditor selects a small number of items to understand a process or identified risk area. Selection is judgmental and area-specific.

## Size determination
n is **specified in the sample MD** for the area being walked through. Typically 1–3 transactions per process. There is no formula.

## When to use
- Planning-stage walkthroughs of significant processes (revenue, purchasing, payroll, treasury, journal entries)
- Identifying control gaps / understanding the flow of transactions
- When the sample MD references this method for an identified-area procedure

## When NOT to use
- Anything intended to support a sampling-based audit conclusion (substantive or controls testing)

## Selection logic
Per the sample MD's identified-area guidance. Typically the auditor traces one transaction from initiation through recording. If the MD specifies multiple selections (e.g., one routine, one non-routine, one period-end), follow that.

## Output
Walkthrough documentation goes in the planning workpaper, not the sample documentation file. This method does not produce a sample pull file.
